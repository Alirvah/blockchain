Sealing a Block — Writing to the Chain

Pending transfers are collected into a block. When the block is sealed, the system computes a SHA-256 hash of everything inside it — every transfer, the previous block's hash, the timestamp, and the nonce. This hash becomes the block's permanent identity.

The Chain — Blocks Linked by Hashes

Each block stores the hash of the previous block. This creates an unbreakable chain: if you tamper with block #2, its hash changes, which means block #3's "previous hash" no longer matches, which breaks block #4, and so on. The entire chain after the tampered block becomes invalid.

Chain Validation — Verifying Integrity

Anyone can verify the chain by recomputing every block's hash from its contents and checking that each block's "previous hash" matches the actual hash of the block before it. If all hashes match, the data has not been tampered with.

The Genesis Anchor — Proof Beyond the Database

Chain validation proves internal consistency, but what if someone replaced the entire database? That's where the genesis anchor comes in. The genesis block's hash is saved in a file (anchors/genesis.json) and committed to a public git repository.

Git commits are cryptographically signed. Anyone can go to the public repository, find the commit that wrote the anchor file, and compare the hash inside it to the hash in the live database. If they match, the genesis block — the root of trust — has not been replaced.

PatCoin goes one step further: the same genesis manifest can also be wrapped in an OpenTimestamps proof file (anchors/genesis.json.ots), which can later resolve to one or more Bitcoin attestations. That gives users a path from the local manifest, to a public Git commit, to a public Bitcoin transaction.

Putting It All Together

The trust model works in layers, from the inside out:

1
Transaction hashes ensure no individual transfer can be modified without detection.
2
Block hashes seal groups of transfers into immutable units. One changed byte changes the hash.
3
Hash-linked chain makes it impossible to tamper with past blocks without breaking all subsequent links.
4
Genesis anchor ties the root of the chain to a public, immutable git commit — proof that extends beyond this server.
